⬆ One-click YouTube upload — setup

FightSync can publish your finished video straight to YouTube — with the title, description, tags and chapters it already generated. This is the one-time setup that turns it on.

✅ First: it's free — you won't be billed

YouTube's upload API doesn't cost money and doesn't even need billing turned on. Instead each setup gets a free daily quota — about 6 uploads per day. Go over it and it simply waits until the next day; you're never charged a cent.

Why a login file instead of a simple "API key"?

Uploading a video means writing to your YouTube channel, so Google requires a proper "allow this app to upload for me" permission (OAuth) — a plain API key only reads public data and can't upload. That's why the steps below create a small credential file instead of a key.

The one-time setup (~5 minutes)

  1. Go to console.cloud.google.com and sign in with the Google account that owns your YouTube channel. (Google-branded pages — not Auth0 or any other "oauth" site.)
  2. Create a project: top bar → project dropdown → New Project → name it "FightSync" → Create → select it.
  3. Enable the API: top search bar → type YouTube Data API v3 → open it → Enable.
  4. Consent screen: left menu → APIs & Services → OAuth consent screen → External → fill App name + your email → on the Test users step, add your own Gmail (this is what lets you use it before Google verifies it).
  5. Create the credential: left menu → APIs & Services → Credentials → + Create Credentials → OAuth client ID → Application type = Desktop app → Create → Download JSON.
  6. Save that file as client_secret.json in your fight-sync folder.
  7. Run once in a terminal: .venv\Scripts\python youtube_auth.py → a browser opens → sign in → if you see "Google hasn't verified this app", click Advanced → Go to FightSync → Allow (it's your own app, so that warning is expected).

Done — reload FightSync and the Export step shows an ⬆ Upload now button. You never repeat this.

⚠ Two things to know

  • Private until verified: while your project is unverified, YouTube may force uploads to Private/Unlisted. Fine for personal use — upload unlisted, then flip to Public in YouTube Studio.
  • Testing-mode tokens: if you leave the consent screen in "Testing", the saved permission expires about weekly and you'd re-run the auth step. Switching it to "Production" (still just for you) removes that.

Sharing FightSync with someone else? bring your own credential

If a friend runs their own copy of FightSync, they should do the steps above with their own Google account — giving them their own separate free quota, completely independent of yours. The credential is just a file in their folder.

Only if everyone uploaded through one instance you host (your client_secret.json) would they share your ~6/day quota — still no cost, but a shared limit, and you'd need Google to verify the app for non-test users. Either way, each person's video always uploads to their own channel (they sign in with their own account); you're only ever the "doorway," never uploading on your account.
