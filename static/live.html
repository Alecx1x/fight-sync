First-time setup — turn on OBS's WebSocket server
  1. OBS is already installed on this PC. Open it.
  2. In OBS: Tools → WebSocket Server Settings → tick Enable WebSocket server (port 4455). Set a password and copy it.
  3. Paste that password below and Save settings.
  4. Build two OBS scenes: one with your Quest cast + webcam (the live look), and later a between-rounds scene.
  5. Add FightSync's overlay: + → Browser → URL …/overlay (transparent skeleton + HUD; fleshed out in Phase 1–3).
📹 Add cameras — phone & more, wireless via NDI
Each camera becomes an OBS source; make a scene per angle and the auto-director below rotates between them.
  1. iPhone: install NDI HX Camera (App Store, ~$20) and open it — it broadcasts your phone's camera over Wi-Fi. (Apple's built-in Continuity Camera is Mac-only, so on Windows you use an app.)
  2. OBS: install the NDI plugin (DistroAV), restart OBS, then + → NDI Source → pick your phone.
  3. Make an OBS scene per angle — e.g. Cam: Front, Cam: Side, Cam: Phone, Gameplay.
  4. More cameras later: more phones = more NDI HX Camera; a real camera = an HDMI→USB capture card (Elgato Cam Link) or an HDMI→NDI encoder. All appear as OBS sources.
Tips: 5 GHz Wi-Fi, PC on Ethernet, phones plugged in. Simpler single-phone alternatives: Camo or EpocCam.

Tick Simulcast above to go live on Twitch and YouTube at once (FightSync runs a local relay that fans your OBS stream out to both — no re-encode). Leave it unticked to stream to the single selected platform. Set OBS's encoder to settings both accept (≈1080p, 6000 kbps, 2s keyframe).
